# GateLM 하루 완성 병렬 개발 인덱스

이 문서는 하루 완성 병렬 개발 문서의 인덱스다.

긴 마스터 문서를 에이전트에게 한 번에 먹이지 않기 위해, 실제 구현 지시서는 머지 단위로 쪼갠다.

## 사용 방법

각 팀원은 자기 작업 머지에 해당하는 문서 하나만 우선 읽는다.

모든 머지 문서에는 다음이 반복되어 있다.

- 공통 계약
- 역할별 읽을 문서
- 역할별 구현 프롬프트
- 머지 후 통과해야 할 테스트 리스트
- 머지 후 Codex에게 검증을 맡길 때 넣을 프롬프트

## 공통 원칙

- A가 사전 문서만 만들고 기다리는 단계는 없다.
- A~E는 Merge 1부터 동시에 구현한다.
- 계약은 각 머지 문서에 이미 포함되어 있다고 보고 구현한다.
- 문서가 너무 길면 해당 머지 문서와 자기 역할 프롬프트만 우선 읽는다.
- API, DB, Event 계약을 바꾸면 PR 본문에 반드시 적는다.
