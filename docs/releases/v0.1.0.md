# GateLM v0.1.0 Release Draft

Release name: **Organization-Based Gateway MVP**

이 문서는 `v0.1.0` GitHub Release를 바로 발행하기 위한 문서가 아니라, 팀 리뷰를 위한 release readiness 초안이다. 실제 tag 생성, GitHub Release publish, shared branch push는 별도 승인 후 진행한다.

## Version 정리

| 구분 | 값 | 의미 |
|---|---|---|
| Latest GitHub Release | `v0.0.1` | 현재 공개 release |
| Next release target | `v0.1.0` | 다음 software release 후보 |
| Gateway spec | `v2.0.0` | 현재 Gateway 계약/schema/fixture version |
| Legacy milestone | `v1.0.0` | historical milestone, 현재 product release가 아님 |
| Future draft | `v2.1.0` | self-host/evaluation draft, 현재 release 계약이 아님 |

## 감사 요약

- Git tag는 `v0.0.1-rc.1`, `v0.0.1`이 확인됐다.
- GitHub Release는 `v0.0.1-rc.1` prerelease(`2026-06-28T16:12:03Z`)와 `v0.0.1` latest(`2026-07-02T05:53:02Z`)가 확인됐다.
- root `package.json`은 private workspace이며 version은 `0.0.0`이다.
- `apps/control-plane-api`, `apps/web`, `apps/application` package version은 `0.1.0`이다.
- `apps/ai-service`는 Python package 기준 `0.0.0`이다.
- pre-cleanup documentation state 보존용 local branch `archive/pre-public-docs-v0.1.0`을 생성했다.

## 문서 구조 변경

- 현재 Gateway 계약, schema, fixture는 `specs/gateway/v2.0.0/`로 분리했다.
- `docs/archive/v1.0.0/`은 legacy milestone으로 낮췄다.
- `docs/archive/gateway-v2.0.0-planning/`은 historical planning으로 낮췄다.
- `docs/drafts/gateway-v2.1.0/`은 future/draft material로 유지한다.
- `README.md`와 `docs/README.md`는 software release version과 gateway spec version을 분리해서 설명한다.

## Release Highlights

- Organization 기반 Gateway MVP의 핵심 경로를 `v0.1.0` release candidate로 설명할 수 있게 정리했다.
- Gateway v2.0.0 계약을 public docs와 분리해 `specs/` 아래 Source Of Truth로 배치했다.
- README를 toy project 설명이 아니라 공개 프로젝트 entry point로 재작성했다.
- archive/draft 문서의 권위를 낮춰 현재 release와 future planning을 구분했다.
- Mock fallback, in-memory/thin-slice, self-host draft, AWS/SaaS 미검증 상태를 숨기지 않고 문서화했다.

## Known Gaps And Limitations

- `v0.1.0`은 production-ready SaaS로 선언하지 않는다.
- `deploy/selfhost/`는 draft이며 검증된 customer-supported self-host release로 선언하지 않는다.
- AWS 배포는 제품 방향이지만 현재 release evidence로 운영 배포를 보장하지 않는다.
- `v2.1.0` self-host/evaluation 문서는 future/draft material이다.
- 일부 historical architecture/reference 문서는 장기 목표와 과거 v1/P0 표현이 섞여 있으며 현재 계약 판단에는 사용하지 않는다.
- root package version은 workspace metadata로 남아 있고 release version과 직접 동기화하지 않았다.
- `LICENSE` 파일이 아직 없다.
